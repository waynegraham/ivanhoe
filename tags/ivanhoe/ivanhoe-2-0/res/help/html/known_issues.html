Known Issues

The following are known issues and limitations with the milestone 4 (July 28, 2004) release of Ivanhoe:

  • The visualization of the player discourse field doesn't show other players within your field
  • All documents in the visualization have equal size. Eventually the quadrant size will reflect the amount of game play.
  • The algorithm for the placement and movement of players within the discourse field will require tuning
  • All documents are treated as one large page. There is no page turner feature at this time.

The following are known issues and limitations with the milestone 3 (June 2, 2004) release of Ivanhoe:

  • The visualization of the discourse field shows your own activity only.
  • The "play" button in the DF visualization isn't operating, but the next and prev buttons work fine
  • There is no support for creating internal links within documents or to create links from one document to another.
  • All documents are treated as one large page.  There is no page turner feature at this time.
  • The comment () feature renders poorly if the user types a long comment.  A long comment would be one with more than one hundred words.  A discussion posting or new document may be more appropriate for such comments.

The following are known issues and limitations with the milestone 2 (April 6) release of Ivanhoe:

  • There is no support for creating internal links within documents or to create links from one document to another.
  • All documents are treated as one large page.  There is no page turner feature at this time.
  • The comment () feature renders poorly if the user types a long comment.  A long comment would be one with more than one hundred words.  A discussion posting or new document may be more appropriate for such comments.
  • The client program can only engage one game at a time
  • Chats are not archived
  • Time stamp for moves is based on client's system clock, so it must be fairly accurate to avoid odd behavior
  • Deleting overlapping links is problematic, the overlap renders incorrectly until restarting the application
  • Copy and paste sometimes inserts some artifacts (control characters) when the text copied is less than a full word.  Using the copy/paste toolbar buttons seems to be more reliable.
